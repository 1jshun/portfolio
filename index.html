건설 현장 운영 데이터를 연결하는 통합 관리 플랫폼

서비스 개요

DnDn은 건설 현장에서 분산되어 관리되던 공정 일정, 인력, 현장 문서 데이터를 하나의 흐름으로 연결해 관리하는 건설 공정 관리 플랫폼입니다. 공정 계획부터 작업 지시, 일일 보고, 공정 분석까지 이어지는 현장 관리 프로세스를 시스템화하여 일정 관리, 인력 배치, 문서 관리 업무를 통합적으로 수행할 수 있도록 지원합니다.

핵심 구현 내용
  1. 일정 관리 API 설계 및 구현 공정 계획, 작업 지시, 일일 보고 등 현장 일정 데이터를 생성·관리하는 API를 설계하고 구현했습니다. 작성된 일정 데이터는 Kafka를 통해 문서 관리 서비스로 전달되어 현장 문서 데이터로 적재되도록 구현했습니다.
  2. 문서 관리 기능 MSA 분리 문서 관리 기능을 별도 MSA 서비스로 분리하고, Gateway와 Kafka 기반 비동기 메시징을 통해 서비스 간 결합도를 낮췄습니다. 이를 통해 문서 처리와 검색 인덱싱 흐름을 독립적으로 확장할 수 있는 구조를 갖췄습니다.
  3. 검색 기능 고도화 Logstash 기반 인덱싱 파이프라인을 구성해 문서 본문과 메타데이터를 Elasticsearch에 적재하고, 문서명 중심이던 기존 검색을 본문·공종·작성일의 복합 조건 기반 검색으로 확장하여 문서 검색 정확도와 성능을 개선했습니다.
  4. Kubernetes 환경에서 서비스 라우팅 및 배포 구조 구성 Kubernetes 환경에서 Frontend, Backend, Gateway, MSA 서비스를 분리 배포하고, Ingress 기반 라우팅과 Blue-Green 배포 구조를 구성했습니다. 이를 통해 기존 백엔드 API와 문서 관리 MSA API의 배포 단위를 분리하고, 신규 버전 배포 시 트래픽 전환과 롤백이 가능한 안정적인 배포 구조를 마련했습니다.
성과
  • 문서 검색 평균 TPS 6.1 → 219.5, 약 36배 향상
  • 문서 검색 평균 응답 시간 9,318ms → 129.98ms, 약 98.6% 단축
  • 오류 건수 590건 → 0건
  • MSA 분리를 통해 문서 관리 기능의 독립성과 유지보수성 개선
  • 파일명 중심 검색에서 본문·공종·작성일 기반 복합 조건 검색으로 확장

POTICARD

개발자의 경험과 역량을 디지털 명함과 포트폴리오로 관리하는 커리어 플랫폼

프로젝트 정보
Period 2026.01.12 — 2026.03.31
Team 4명 / 팀장
Role 포트폴리오 관리 API 설계 및 구현, AI 첨삭 및 기술 스택 추출 기능 구현, 결제 검증 및 요금제 구독 관리 기능 구현
서비스 개요

POTICARD는 사용자의 경력과 프로젝트 경험을 포트폴리오와 디지털 명함 형태로 정리하고 공유할 수 있도록 지원하는 커리어 관리 플랫폼입니다. 사용자는 자신의 경험을 포트폴리오로 작성하고, AI 기반 첨삭과 기술 스택 추출 기능을 통해 핵심 역량을 더 명확하게 정리할 수 있습니다. 또한 요금제 구독을 통해 AI 첨삭 등 고급 기능을 사용할 수 있도록 결제 및 권한 관리 흐름을 함께 제공합니다.

핵심 구현 내용
  1. 포트폴리오 API 설계 및 구현 포트폴리오를 섹션 단위로 관리하는 API 구조를 설계하고, 목록 조회에 페이징을 적용해 응답 데이터 크기를 관리했습니다.
  2. 이미지 업로드 구조 개선 서비스 내 이미지 업로드 기능을 S3 기반으로 통합했습니다. 서버에 파일을 직접 저장하지 않고 S3 URL을 데이터로 관리하여 애플리케이션 서버의 파일 I/O 부담을 줄이고, 배포 환경이 변경되어도 이미지 접근 경로가 유지되도록 설계했습니다.
  3. 결제 위변조 검증 로직 구현 클라이언트에서 결제 완료 후 전달한 결제 정보를 서버에서 PortOne API로 재조회하도록 구현했습니다. 결제 상태와 실제 결제 금액을 검증하고, 요청 금액과 PG사 결제 금액이 일치하지 않을 경우 예외를 발생시켜 결제 위변조 가능성을 차단했습니다.
  4. 요금제 기반 기능 접근 제어 로직 구현 사용자의 최근 결제 내역을 기준으로 PRO 요금제 활성화 여부를 판단하는 로직을 구현했습니다. 이를 바탕으로 AI 첨삭과 같은 유료 기능 접근을 제어하여 구독 상태에 따라 기능 사용 가능 여부가 달라지도록 구성했습니다.
성과
  • 포트폴리오 목록 조회에 페이징 전략 도입으로 MTT 29.7ms → 19.5ms 단축, TPS 65.5 → 100.9, 51.7% 향상
